Convergence
Narrows rails through the merged phase of the loop (0 = constant width, 1 = ≈15% width at merge).
Cross-fades non-trunk rails toward the trunk's colour as they approach the merge. 0 = stays green/yellow distinct; 1 = greens fully blend to yellow at the junction.
Spreads the colour transition across neighbouring segments. 0 = blend only inside the bend; 1 = blend bleeds ≈6 segments to either side, so colours start fusing well before the junction.
Shared shade every rail washes toward at the junction. Pick something close to the background (default #C9C6BC) so rails dissolve into a pale ribbon at the merge instead of one colour visibly overtaking another.
When on, rail 0 (the trunk) keeps its base colour through the entire cycle — only branches wash toward the merge tint. Matches Railway's brand reading: trunk = always-shipped mainline; branches = active work calming to the shipped state at the merge.
Shape of the colour wash inside a bend. 1 = linear (cream only fully visible at the bend's end). < 1 = easeOut (cream readable immediately the rail starts curving — calmness arrives early). > 1 = easeIn (cream held back until the bend completes).
When on, rail 0 (the always-present trunk) composites over the split rails — greens recede behind the yellow at the merge instead of covering it.
When on, all rails are composited as a single MAX-alpha shape (winner-takes-all colour) instead of per-rail alpha-blending. Removes the internal "seams" that show up where two soft-edged rails overlap, so where the rails converge they read as one continuous fused body. In parallel sections (no overlap) the rails still appear separate. Overrides Trunk on top.
Fades non-trunk rails to transparent at the junction so they "grow out of" the trunk gradually instead of popping in as a sharp wedge. Combine with Blend softness to spread the fade across multiple segments.
Pulls non-trunk rails toward the trunk's lane by the softened phase, so the visible bend stretches across many segments instead of being confined to the one bend segment. Raise Blend softness to widen the zone the pull operates over.
Width multiplier for non-trunk rails. 1 = same width as the trunk; 2.5–3 = wider, softer "halo" branches that match the figma-svg merge look (sharp saturated spine + blurred pale fade-ins). Trunk width is untouched.

Rail colors
Profile
Shoulder colors (per rail)
Figma mode replaces the Gaussian halo with a 6-stop vertical gradient (transparent edge → pastel shoulder → saturated core, mirrored). Core / Shoulder are normalized distances from rail centre (0=centre, 1=edge).
Colour timeline
When on, every rail's body colour follows keyframes anchored to the loop's normalised position (0..1). Stops are lerped between. The merge wash still applies on top. Per-rail offsets stagger the colour cascade so e.g. the trunk reaches blue before the branches.
Container hull
Saturated ribbon rendered behind the rails. Mirrors the rail topology with each non-trunk rail's centerline pushed outward by Offset, so the hull's funnel flares slightly wider than the rails' funnel (matches svg/containers/v2/container.svg). The trunk lane is never offset.
Ticks (zoetrope)
0 = crisp ticks (may strobe off the freeze speed at high scroll rates); 1 = ticks widen to cover one frame's camera travel and become soft non-flicker bars. Use "Snap to freeze speed" below for the crispest result.
freeze ≈ — wu/s @ 60 Hz
0 = use the display's detected refresh rate (auto). Any positive value locks the render loop to that fps via frame-skipping — useful on 120 Hz / ProMotion displays where you want deterministic tick freeze at 60 fps. Snap-to-freeze multiplies by this rate.
Rounds the camera's per-frame step to a whole-pixel multiple so ticks land at the same sub-pixel offset every frame — kills the temporal-aliasing flicker on narrow ticks at any speed. A fractional accumulator preserves average speed exactly. Recommended on; turn off only if you're testing extremely slow scroll speeds where the 1-pixel quantum is visible.
Bursts
Painterly slugs of colour scrolling along each rail. Each rail picks its own bursts and drifts at its own velocity (Rail spread = 0 → all rails identical, 1 → rails desync hard). Lock to rail flow swaps to mode B: bursts pinned to world-X with per-rail parallax instead.
